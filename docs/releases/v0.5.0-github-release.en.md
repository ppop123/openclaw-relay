# OpenClaw Relay v0.5.0

v0.5.0 fixes stability issues in agent-to-agent communication and enforces product boundaries at the code level.

## Bug Fixes

- **Peer streaming timeout** — Requests no longer hang when the remote sends a stream-end frame without a final response
- **Redial replay** — Reconnecting to a peer no longer replays chunks that were already delivered
- **Signal stealing** — Auto-accept no longer consumes non-invite signals, so manual and automatic connection flows no longer interfere with each other

## Security Hardening

- All `relay.peer.*` methods are now guarded to agent/operator only; browser clients cannot invoke them
- Concurrent dial attempts are coalesced, and redial uses backoff

## Documentation

- Quick Start rewritten with reproducible step-by-step instructions
- `relay.peer.request` semantics clarified as "queued to relay", not "delivered"
