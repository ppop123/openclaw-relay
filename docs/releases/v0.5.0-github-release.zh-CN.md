# OpenClaw Relay v0.5.0

`OpenClaw Relay v0.5.0` 现已发布。

这是一个以 **agent-only peer capability 耐用性补强**、**peer 方法边界收紧**、以及 **文档/使用路径更可复现** 为重点的小版本发布。

## 重点更新

- 修复 peer streaming 在只收到 `stream_end` 而没有 trailing `response` 时可能挂到超时的问题
- 修复 peer stream redial 在已经输出部分 chunk 后可能重放重复内容的问题
- 修复 peer auto-accept 会吞掉非 `invite_request` signal 的问题，避免手动 dial / connect 与自动处理互相踩踏
- 为所有 `relay.peer.*` gateway 方法增加代码级 **agent-only / operator-only** 守卫，浏览器类 human-facing client 不再能调用这些方法
- 合并并发 dial、补上 redial backoff，并把 `relay.peer.request` 的返回语义改成“已排队到 relay”，不再假装已送达
- Quick Start、README、source-first 开发要求和 soak 脚本一起收口，减少误用和不可复现步骤

## 正式支持范围

`v0.5.0` 的正式支持范围保持不变：

- `relay/` — Go relay 服务
- `sdk/python/` — Python 客户端 SDK（Layers 0–2）
- `client/` — Web 参考客户端
- `plugin/` — OpenClaw 网关插件
- `protocol/` — 协议文档、错误码、canonical fixtures

## 不在正式支持范围内

以下内容仍然不属于正式支持面：

- `sdk/js/` — 尚未实现
- 超出当前文档定义的未来协议扩展能力

## v0.5.0 具体变化

- 这是一个小版本发布，不扩大正式协议面，但显著提高了 agent-only peer capability 的耐用性
- `relay.peer.*` 现在在代码层真正落实了“agent 可以找 agent，人类客户端不能越权”的产品边界
- README / Quick Start 现在明确说明：先 `enable`，再在 `pair --wait` 窗口里完成浏览器连接
- OpenClaw 相关开发要求也继续收严：涉及 OpenClaw 运行时行为时，必须先看本机源码，不能靠猜
- `v0.5.0` 建议作为当前 `v0.5.x` 线上的推荐版本使用

## 验证结果

`v0.5.0` 已通过：

- `cd relay && go test -v -count=1 ./...`
- `cd sdk/python && pytest -q`
- `cd client && npm test`
- `cd plugin && npm test`
- `cd plugin && npm run typecheck`
- `cd plugin && npm run build`
- `python3 scripts/validate-protocol-examples.py`
- `bash scripts/check-doc-consistency.sh`

## 说明

- `v1` 仍然**不提供** forward secrecy
- 浏览器端的 gateway 验证模型是 **user-supplied pinned public key**，不是自动 TOFU
- agent-only peer capability 仍然必须保持产品边界：human-facing client 只能连接自己的 OpenClaw，不能发现或联系别人的 OpenClaw
