# OpenClaw Relay v0.5.0

v0.5.0 集中修复了 agent 间通信的稳定性问题，并在代码层落实了产品边界。

## Bug 修复

- **peer 流式响应超时** — 当对端只发送了结束帧而没有最终响应时，请求不再挂起
- **重连重播** — peer 断线重连后不再重复推送已经输出过的内容
- **信号抢占** — 自动接受连接不再吞掉其他类型的信号，手动连接和自动连接不再互相干扰

## 安全加固

- 所有 `relay.peer.*` 方法现在由代码守卫限制为仅 agent/operator 可调用，浏览器客户端无法越权访问
- 并发拨号请求自动合并，重连加入退避间隔

## 文档改进

- Quick Start 流程重写，步骤可直接复现
- `relay.peer.request` 语义明确为"已提交到中继"而非"已送达"

完整支持范围和安全模型说明见 [v0.3.0 发布说明](https://github.com/ppop123/openclaw-relay/releases/tag/v0.3.0)。
