# OpenClaw Relay v0.2.1

`OpenClaw Relay v0.2.1` is now available.

This is a patch release focused on **gateway plugin stability**, **real-host lifecycle validation**, and **release hardening** after `v0.2.0`.

## Highlights

- Fixes `openclaw relay pair` completion detection so the CLI reports pairing success reliably
- Adds a real-host lifecycle smoke flow covering:
  - initial pair
  - request validation
  - revoke
  - re-pair
  - rotate-token
  - disable
- Improves local smoke isolation by using a fresh isolated OpenClaw state directory on each run
- Adds a host-bridge regression test for the pairing completion path

## Official Release Scope

The official support surface of `v0.2.1` remains:

- `relay/` — Go relay server
- `sdk/python/` — Python client SDK (Layers 0–2)
- `client/` — Web reference client
- `plugin/` — OpenClaw gateway plugin
- `protocol/` — protocol docs, error codes, canonical fixtures

## Not Included in the Official Release Scope

The following are still outside the official support surface:

- `plugin/` notify-style extensions beyond the documented Layer 3 surface
- `sdk/js/` — not yet implemented

## What Changed in v0.2.1

- Hardened plugin CLI behavior around pairing state detection
- Expanded local release-manager validation from a single happy-path smoke to a lifecycle smoke
- Kept release contracts, protocol examples, and docs consistency checks green after the plugin changes

## Validation

`v0.2.1` passed:

- `cd relay && go test -v -count=1 ./...`
- `cd sdk/python && pytest -q`
- `cd client && npm test -- --run`
- `cd plugin && npm ci && npm test && npm run typecheck`
- `python3 scripts/validate-protocol-examples.py`
- `bash scripts/check-doc-consistency.sh`
- `bash scripts/smoke-openclaw-plugin.sh` (local/manual, real OpenClaw runtime)

## Notes

- `v1` still does **not** provide forward secrecy
- The plugin lifecycle smoke is intentionally local/manual because hosted CI does not ship with an OpenClaw runtime

Feedback, issues, and audits are welcome.
