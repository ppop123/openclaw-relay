# OpenClaw Relay v0.3.0

`OpenClaw Relay v0.3.0` is now available.

This is a feature release focused on **web-client productization**, **persistent browser identity**, **real-browser E2E coverage**, and **protocol/security consistency hardening** after `v0.2.1`.

## Highlights

- The web client now persists its browser identity so a normal reload keeps the same fingerprint
- Adds passphrase-protected identity export/import, fingerprint/public-key copy actions, and clearer recovery hints
- Adds saved relay profiles, preferred-agent memory, a session diagnostics bar, and current-chat export
- Adds both deterministic browser E2E and real relay/gateway live browser E2E coverage
- Hardens the browser transport lifecycle: fixes connect/disconnect concurrency and cleanup, handshake waiter cleanup, and runtime `channelToken` clearing
- Completes the `docs/web-client.md` package and machine-readable truth sources with aligned semantics
- Converts the project homepage into a bilingual README with Chinese first and English second

## Official Release Scope

The official support surface of `v0.3.0` is:

- `relay/` — Go relay server
- `sdk/python/` — Python client SDK (Layers 0–2)
- `client/` — Web reference client
- `plugin/` — OpenClaw gateway plugin
- `protocol/` — protocol docs, error codes, and canonical fixtures

## Not Included in the Official Release Scope

The following remain outside the official release scope:

- `sdk/js/` — not yet implemented
- future protocol extensions beyond the currently documented surface

## What Changed in v0.3.0

- Upgrades the browser identity model from page-memory-only behavior to IndexedDB-backed persistence with strict separation between durable identity and non-persisted secrets
- Adds direct tests and E2E coverage for identity import/export, recovery, pinned gateway key verification, and reload/reconnect behavior
- Aligns protocol and security docs around explicit pinned-key verification instead of TOFU wording
- Ships a bilingual README homepage as the new project front door

## Validation

`v0.3.0` passed:

- `cd relay && go test -v -count=1 ./...`
- `cd sdk/python && pytest -q`
- `cd client && npm test`
- `cd client && npm run test:e2e`
- `cd client && npm run test:e2e:live`
- `cd plugin && npm test`
- `cd plugin && npm run typecheck`
- `python3 scripts/validate-protocol-examples.py`
- `bash scripts/check-doc-consistency.sh`

## Notes

- `v1` still does **not** provide forward secrecy
- The browser gateway verification model is **user-supplied pinned public key verification**, not automatic TOFU
- Browser identity persistence depends on IndexedDB; blocked environments fall back to a page-scoped identity

Feedback, issues, and audits are welcome.
