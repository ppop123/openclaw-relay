# OpenClaw Relay v0.2.0

今天我们发布 `OpenClaw Relay v0.2.0`。

这是一个面向 OpenClaw 远程连接场景的开源 relay 方案：在不暴露公网 IP、不依赖第三方聊天平台的前提下，让客户端和本地 OpenClaw 网关通过 relay 建立端到端加密连接。

## 本次发布包含什么

本次 `v0.2.0` 的**正式支持范围**：

- `relay/` — Go 中继服务
- `sdk/python/` — Python 客户端 SDK（协议 Layers 0–2）
- `client/` — Web 参考客户端
- `protocol/` — 协议文档、机器可读真相源、canonical fixtures
- `plugin/` — OpenClaw 网关插件

**不在本次正式支持范围内**：

- `deploy/cloudflare-worker/` — 实验性实现，不属于正式发布面
- `sdk/js/` — 尚未实现

## 当前能力

`v0.2.0` 已提供以下能力：

- WebSocket relay 转发
- Layer 0 / 1 / 2 协议分层
- 端到端加密传输
- 用户提供的 pinned 网关公钥校验
- payload 限制、rate limit、Origin 校验
- Python / Web / plugin 测试覆盖
- 协议示例与文档一致性校验
- AI-first 机器可读真相源：
  - `docs/support-matrix.json`
  - `docs/release-manifest.json`
  - `protocol/error-codes.json`
  - `protocol/examples/*.json`

## 安全边界

当前版本的安全边界如下：

- relay 无法读取应用层明文
- 使用 X25519 + HKDF-SHA256 + AES-256-GCM
- 每个连接使用新的 session nonce 派生新的会话密钥
- 支持 nonce 方向校验与重放保护
- 浏览器侧网关身份校验使用用户提供的 pinned 公钥

同时必须明确：

- `v1` **不提供 forward secrecy**
- Web 参考客户端的身份 keypair 当前是**内存态**：
  - 同一页面会话内可复用
  - 页面刷新后会重新生成
- Cloudflare Worker 版本仍为实验实现，不建议生产使用

## 发布质量

在本次发布前，仓库通过了以下 release gates：

- `cd relay && go test -v -count=1`
- `cd sdk/python && pytest -q`
- `cd client && npm test`
- `cd client && npm ci && cd .. && client/node_modules/.bin/vitest run plugin/tests`
- `cd deploy/cloudflare-worker && npm ci && cd ../.. && deploy/cloudflare-worker/node_modules/.bin/tsc -p plugin/tsconfig.json --noEmit`
- `python3 scripts/validate-protocol-examples.py`
- `bash scripts/check-doc-consistency.sh`

这意味着当前仓库的实现、测试、文档、机器可读契约已经完成一轮针对“公开发布 + AI-first”的收口审查。

## 适合谁

这个版本适合：

- 想自托管 OpenClaw relay 的开发者
- 想基于开放协议实现自定义客户端的开发者
- 想把 relay 能力装进自己 OpenClaw 运行时的开发者
- 想研究 AI-first 协议/中继设计的工程团队

如果你需要的是：

- 一键开箱即用的完整产品体验
- 多节点、高可用、生产级控制面

那么这些还不在 `v0.2.0` 范围内。
