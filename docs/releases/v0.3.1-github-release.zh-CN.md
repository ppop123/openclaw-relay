# OpenClaw Relay v0.3.1

`OpenClaw Relay v0.3.1` 现已发布。

这是一个小型 hotfix 版本，重点是收口 `v0.3.0` 之后发现的 **plugin 真实 relay 集成测试稳定性问题**，并把首页上的“怎么使用”入口写得更直接。

## 重点更新

- 修复 `plugin` 真实 relay 集成测试的工作目录问题，避免 CI 在 `working-directory: plugin` 下错误解析 relay 路径
- 将该测试从 `go run` 冷启动改为“先构建临时 relay 二进制再启动”，避免把 Go 冷编译时间误算进 ready timeout
- 在 README 首页新增更明确的“如何使用 / How To Use It”入口，直接说明 relay、plugin、web client 的上手路径

## 正式支持范围

`v0.3.1` 的正式支持范围保持不变：

- `relay/` — Go relay 服务
- `sdk/python/` — Python 客户端 SDK（Layers 0–2）
- `client/` — Web 参考客户端
- `plugin/` — OpenClaw 网关插件
- `protocol/` — 协议文档、错误码、canonical fixtures

## 不在正式支持范围内

以下内容仍然不属于正式支持面：

- `sdk/js/` — 尚未实现
- 超出当前文档定义的未来协议扩展能力

## v0.3.1 具体变化

- 不改变用户侧协议面和核心使用方式，主要修复发布后暴露的 plugin CI / 集成测试脆弱点
- README 首页现在明确告诉新用户应该如何部署 relay、安装 plugin、再通过浏览器连接
- `v0.3.1` 建议作为 `v0.3.x` 线上的推荐版本使用

## 验证结果

`v0.3.1` 已通过：

- `cd relay && go test -v -count=1 ./...`
- `cd sdk/python && pytest -q`
- `cd client && npm test`
- `cd client && npm run test:e2e`
- `cd client && npm run test:e2e:live`
- `cd plugin && npm test`
- `cd plugin && npm run typecheck`
- `python3 scripts/validate-protocol-examples.py`
- `bash scripts/check-doc-consistency.sh`

## 说明

- `v1` 仍然**不提供** forward secrecy
- 浏览器端的 gateway 验证模型是 **user-supplied pinned public key**，不是自动 TOFU
- 浏览器身份持久化依赖 IndexedDB；不可用时会回退为当前页面会话内身份
