# OpenClaw Relay v0.1.0

Today we are releasing `OpenClaw Relay v0.1.0`.

This is an open-source relay solution for remote OpenClaw connectivity: it lets a client and a locally running OpenClaw gateway establish an end-to-end encrypted connection without exposing a public IP and without relying on third-party chat platforms.

## What is included in this release

The **officially supported scope** of `v0.1.0` includes:

- `relay/` — Go relay server
- `sdk/python/` — Python client SDK (protocol Layers 0–2)
- `client/` — Web reference client
- `protocol/` — Protocol documentation, machine-readable truth sources, and canonical fixtures

**Not included in the official release scope**:

- `deploy/cloudflare-worker/` — experimental implementation, excluded from the official release surface
- `plugin/` — not yet implemented
- `sdk/js/` — not yet implemented

## Current capabilities

`v0.1.0` currently provides:

- WebSocket relay forwarding
- Layer 0 / 1 / 2 protocol separation
- End-to-end encrypted transport
- TOFU-based gateway identity verification
- payload limits, rate limits, and Origin validation
- Python / Web client test coverage
- protocol example validation and documentation consistency checks
- AI-first machine-readable truth sources:
  - `docs/support-matrix.json`
  - `docs/release-manifest.json`
  - `protocol/error-codes.json`
  - `protocol/examples/*.json`

## Security boundaries

The current security boundary is:

- the relay cannot read application plaintext
- X25519 + HKDF-SHA256 + AES-256-GCM are used
- each connection derives a unique session key from fresh session nonces
- nonce direction checks and replay protection are enforced
- gateway identity verification follows a TOFU model

It is equally important to be explicit about the current limitations:

- `v1` does **not** provide forward secrecy
- the web reference client currently keeps its identity keypair **in memory only**:
  - stable within a single page session
  - regenerated after page reload
- the Cloudflare Worker version remains experimental and is not recommended for production use

## Release quality gates

Before this release, the repository passed the following release gates:

- `cd relay && go test -v -count=1`
- `cd sdk/python && pytest -q`
- `cd client && npm test`
- `python3 scripts/validate-protocol-examples.py`
- `bash scripts/check-doc-consistency.sh`

This means the current implementation, tests, documentation, and machine-readable contracts have gone through a dedicated public-release + AI-first consistency review.

## Who this release is for

This release is a good fit for:

- developers who want to self-host an OpenClaw relay
- developers who want to build their own clients on top of an open protocol
- engineering teams interested in AI-first protocol and relay design

If you need:

- a fully integrated out-of-the-box product experience
- a completed gateway plugin
- multi-node, HA, production-grade control-plane features

those are not part of `v0.1.0`.

## What comes next

Likely next priorities include:
