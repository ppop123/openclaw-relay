# OpenClaw Relay v0.3.0

`OpenClaw Relay v0.3.0` 现已发布。

这是一个以 **Web Client 产品化**、**浏览器身份持久化**、**真实浏览器 E2E** 和 **协议/安全口径收口** 为重点的功能版本，承接 `v0.2.1`。

## 重点更新

- Web Client 现在支持浏览器身份持久化，正常 reload 后保持同一指纹
- 新增受口令保护的身份导出 / 导入、指纹与公钥复制、身份恢复提示
- 新增保存的 Relay profile、preferred agent 记忆、会话诊断栏、当前聊天导出
- 补齐 deterministic browser E2E 与 real relay/gateway live browser E2E
- 收紧 browser transport 生命周期：修复 connect/disconnect 并发与清理问题、握手等待器清理、`channelToken` 运行时清理
- 完成 `docs/web-client.md` 文档包和 machine-readable truth sources 的一致性收口
- 将项目首页改为双语结构，中文在前、英文在后

## 正式支持范围

`v0.3.0` 的正式支持范围：

- `relay/` — Go relay 服务
- `sdk/python/` — Python 客户端 SDK（Layers 0–2）
- `client/` — Web 参考客户端
- `plugin/` — OpenClaw 网关插件
- `protocol/` — 协议文档、错误码、canonical fixtures

## 不在正式支持范围内

以下内容仍然不属于正式支持面：

- `sdk/js/` — 尚未实现
- 超出当前文档定义的未来协议扩展能力

## v0.3.0 具体变化

- 浏览器客户端身份从“页面内临时身份”升级为“IndexedDB 可持久化身份 + 不可持久化 secret 严格分离”
- 为身份导入导出、错误恢复、pinned gateway key 校验、浏览器 reload / reconnect 行为补齐单测与 E2E
- 协议与安全文档统一改为 explicit pinned-key 模型，而不是 TOFU
- 新增 bilingual README 首页，作为新的项目主页入口

- `cd sdk/python && pytest -q`
- `cd client && npm test`
- `cd client && npm run test:e2e`

- `cd plugin && npm test`

- `python3 scripts/validate-protocol-examples.py`
- `bash scripts/check-doc-consistency.sh`

## 说明

- `v1` 仍然**不提供** forward secrecy
- 浏览器端的 gateway 验证模型是 **user-supplied pinned public key**，不是自动 TOFU
- 浏览器身份持久化依赖 IndexedDB；不可用时会回退为当前页面会话内身份
