# OpenClaw Relay v0.8.0

v0.8.0 ships the long-awaited JavaScript SDK, promotes the desktop shell to an official release gate, and adds short-lived admin keys for the management UI.

## JavaScript SDK (Node)

- New `sdk/js/` package with a Node-focused Relay client (Layers 0–2)
- X25519 + HKDF + AES-256-GCM handshake and transport parity with the browser reference client
- Streaming `chat.send`, `agents.list`, and `system.status` support
- Vitest coverage for crypto and transport behaviors

## Desktop Shell

- Desktop shell is now officially supported and included in release gates
- Shared web client assets sync into `desktop/webview-dist/` via `npm run sync-client`
- Release build uses `tauri build --bundles app` for local app artifacts

## Admin UI Hardening

- Admin panels now require a short-lived admin key in addition to an approved client
- New `/relay_admin` chat command and `openclaw relay admin-key` CLI helper
- Admin key is never persisted in the browser client and can be cleared at any time

---

For the full support scope and security model, see the [v0.3.0 release notes](https://github.com/ppop123/openclaw-relay/releases/tag/v0.3.0).
