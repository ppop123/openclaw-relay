# OpenClaw Relay v0.3.1

`OpenClaw Relay v0.3.1` is now available.

This is a small hotfix release focused on **plugin real-relay integration-test stability** after `v0.3.0`, plus a clearer “how to use it” entry on the project homepage.

## Highlights

- Fixes the `plugin` real-relay integration test so it resolves the relay repo correctly under `working-directory: plugin`
- Changes that test from cold `go run` startup to “build a temporary relay binary first, then launch it”, so Go compile time no longer trips the ready timeout
- Adds a clearer “How To Use It” section to the README homepage so new users can see the relay + plugin + browser flow immediately

## Official Release Scope

The official support surface of `v0.3.1` is unchanged:

- `relay/` — Go relay server
- `sdk/python/` — Python client SDK (Layers 0–2)
- `client/` — Web reference client
- `plugin/` — OpenClaw gateway plugin
- `protocol/` — protocol docs, error codes, and canonical fixtures

## Not Included in the Official Release Scope

The following remain outside the official release scope:

- `sdk/js/` — not yet implemented
- future protocol extensions beyond the currently documented surface

## What Changed in v0.3.1

- Does not change the user-facing protocol surface or core usage model; it mainly fixes a release-follow-up CI and integration-test weakness in the plugin path
- The README homepage now explicitly tells new users how to deploy a relay, install the plugin, and connect from the browser
- `v0.3.1` is the recommended `v0.3.x` patch release going forward

## Validation

`v0.3.1` passed:

- `cd relay && go test -v -count=1 ./...`
- `cd sdk/python && pytest -q`
- `cd client && npm test`
- `cd client && npm run test:e2e`
- `cd client && npm run test:e2e:live`
- `cd plugin && npm test`
- `cd plugin && npm run typecheck`
- `python3 scripts/validate-protocol-examples.py`
- `bash scripts/check-doc-consistency.sh`

## Notes

- `v1` still does **not** provide forward secrecy
- The browser gateway verification model is **user-supplied pinned public key verification**, not automatic TOFU
- Browser identity persistence depends on IndexedDB; blocked environments fall back to a page-scoped identity

Feedback, issues, and audits are welcome.
